# Create Documentation

After creating a component in the "ui" package, always create a corresponding docs MDX file with the structure in the provided template. Comments in this example are instructions for you to fill in information.

The props and slots section formatting should be strictly adhered to. Other sections such as the variants can be a little looser, but should generally aim to provide developers with a concise example of the component's features. Code examples are encouraged wherever relevant.

## Steps

1. Review the component's .tsx files and analyze how it functions
2. Fill in the component's .mdx documentation according to the template.
